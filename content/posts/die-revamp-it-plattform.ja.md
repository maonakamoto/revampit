まだ動くコンピューターがコンテナに捨てられる。ちょうどそのとき、数本先の通りに、まさにそのコンピューターを必要としているのに新しい機器を買う余裕のない人が座っている。Revamp-IT では長年、この二人のあいだに橋はなく、互いに口をきかないソフトウェアの継ぎはぎがあるだけでした。私たちはこの隙間を埋めました — 寄付された古い機器から新しい持ち主までの全行程を映し出す、ひとつのプラットフォームで。

## 問題:電子廃棄物と、沈黙する3つのシステム

電子廃棄物は世界で最も急速に増えている廃棄物の流れのひとつです。捨てられる機器の多くは壊れているわけではなく — ただ古く、遅く、あるいは流行遅れなだけです。少しの手入れと SSD、そして軽量な Linux があれば、それらは再び立派な作業機器に戻ります。まさにそれが私たちの使命です:中古のコンピューターは修理され、次の持ち主へと渡される — 埋め立てられるのではなく。

ネックは修理ではなく、その背後にある組織の側にありました。長年のあいだに Revamp-IT では、それぞれ独自の目的を果たし、互いを無視する3つのばらばらな旧システムが育っていました:

- `revamp-it.ch` 上の **Joomla のウェブサイト** — 公の顔でありながら静的で、他から切り離されていました。
- `shop.revamp-it.ch` 上の **Shopware のショップ** — リファービッシュ機器の販売、独自のデータを持つ独立した宇宙。
- `kivitendo.ch` 上の **Kivitendo の ERP** — 社内の会計と在庫管理、これもまた隔離されていました。

この3つのシステムは互いに口をききませんでした。登録され、検査され、保管され、最終的に販売される機器は、各ステーションで手作業で入力し直さなければなりませんでした。データはサイロのあいだを行ったり来たりコピーされ、手作業の転記が伴うあらゆるミスとともにありました。そして本当に大切な人々 — 手頃な技術を必要とする人々 — は、この混乱のなかでほとんど道を見つけられませんでした。循環型経済の非営利団体を訪れる人が、手頃なノートパソコンを手に入れるために3つのウェブサイトを探し回らなければならないなんて、あってはなりません。

## 解決策:公開と社内を同時に担う、ひとつのアプリ

3つの旧システムをこれ以上ガムテープでつなぎとめる代わりに、私たちはひとつのまとまったプラットフォームを築きました。それは両方の世界をカバーします:コミュニティが閲覧し、購入し、助けを求め、私たちのインパクトについて読む**公開側**と、チームが機器を登録し、検査し、公開し、自らの時間を記録し、連携する**社内側**です。両方の世界は同じデータ基盤を共有します。機器はちょうど一度だけ登録され、そこから自動的にショーウィンドウまで進みます。

この記事の残りは、主要な領域をめぐる案内です。それぞれについて、**何をするのか**、そして**どの現実の問題を解くのか**を手短に述べます。

## マーケットプレイス:すべての人のためのショーウィンドウ

![フィルター、CO₂バー、出品のあるマーケットプレイス](/blog/showcase-revampit-marketplace.png)
*フィルター、透明性のある CO₂ バー、そしてリファービッシュ品と個人出品が混在するマーケットプレイス。*

**何をするのか:** マーケットプレイスは、ひとつのショーウィンドウに2つの経路を統合します。ひとつは、Revamp-IT が私たちの工房で整備したリファービッシュ機器。もうひとつは、自分のハードウェアを譲りたい人々の個人出品 — 古典的な個人間取引です。ひとつのカート、ひとつの決済、その背後には Payrexx によるエスクロー機能付きの決済処理があり、買い手と売り手が互いを信頼できるようにします。

**どの問題を解くのか:** かつては販売と個人間の譲渡は別々の片隅に住んでいました、後者がそもそも存在した場合ですが。いまや機器を探す人は、検査済みのリファービッシュの提供も、コミュニティが提供するものも、一か所で見つけられます。フィルターとはっきり見える CO₂ バーが、最も適していて生態学的にも最も理にかなった機器をすばやく見つける助けになります。

## IT ヘルプ:修理と技術者のネットワーク

![修理と技術者のネットワークのある IT ヘルプ](/blog/showcase-revampit-it-hilfe.png)
*IT ヘルプ:修理が必要な人は、ここでネットワークの技術者を見つけられます。*

**何をするのか:** IT ヘルプは、修理の必要がある人々と、助けられる技術者を結びつけます。問題のある人はそれを説明し、技能のある人は名乗り出ます。需要と供給が互いを見つける、修理のための双方向ネットワークです。

**どの問題を解くのか:** 修理の知識は分散していて、たどり着くのが難しいことが多いものです。壊れたノートパソコンを再び動かせる人を、誰もが知っているわけではありません。ネットワークはこの知識を可視化し、手の届くものにします — そうしてさらに多くの機器を、買い替えるのではなく生かし続けます。成功した修理のひとつひとつが、廃棄物のなかの機器をひとつ減らします。

## サービス:あなたのコンピューターを組む、Linux、そしてリサイクル

![サービス「あなたのコンピューターを組む」](/blog/showcase-revampit-service.png)
*サービス「あなたのコンピューターを組む」— 自分のニーズに合わせてハードウェアを組み立てる。*

**何をするのか:** 単なる機器販売にとどまらず、プラットフォームは私たちのサービスをまとめています。「あなたのコンピューターを組む」は、関心のある人が自分のニーズに合わせて機器を組み立てられるようにします。そこに、Linux とオープンソースソフトウェアをめぐる相談と導入、さらに本当にもう救えないもののための適正なリサイクルが加わります。

**どの問題を解くのか:** 誰もが同じものを必要としているわけではありません。完成したノートパソコンが欲しい人もいれば、あつらえの作業機器が欲しい人もいれば、プロプライエタリなオペレーティングシステムから離れて自由なソフトウェアへ向かいたい人もいます。そして機器が寿命を迎えたら、正しく環境に配慮して活用されるべきです。この幅広さを一か所に持つことこそ、多くの人にとって循環型経済を初めて実行可能にするものです。

## CO₂ インパクト:主張ではなく、裏付け

![透明性ページ上の CO₂ 方法論](/blog/showcase-revampit-co2.png)
*透明性ページ上の CO₂ 方法論 — すべての数字が、その導出を開示して示されます。*

**何をするのか:** プラットフォーム上のすべての CO₂ の記載には、たどれる根拠があります。係数は ADEME のオープンデータに由来し、Fraunhofer と ZHAW の研究と照合されます。透明性ページでは方法論が完全に開示されています:どのように計算されるのか、その背後にどの出典があるのか、そして信頼できる値が存在しないためにあえて主張を控えるのはどこか。

**どの問題を解くのか:** 持続可能性の数字は、主張するのは安く、裏付けるのは高くつきます。グリーンウォッシングは、まさにそれが促進すべき運動そのものへの信頼を損ないます。すべての数字を出典に結びつけ、計算を公開することで、私たちのインパクトは検証可能なままにとどまります — 買い手にとって、パートナーにとって、そして私たち自身にとって。

## 機器の受け入れ:AI 登録と四つ目の原則

**何をするのか:** 社内の機器受け入れでは、寄付または購入されたすべての機器が登録されます。AI 支援の登録が写真とデータを取り込み、カテゴリーと主要データを提案します。検査が義務づけられたカテゴリーの機器は、チェックリストを伴う構造化された品質管理を通ります。重要なステップは四つ目の原則に従います。より大量の場合には、一括登録と CSV による取り込みがあります。

**どの問題を解くのか:** まさにここに、旧世界の最大の摩擦損失がありました — ショップと ERP のあいだの果てしない再入力です。いまや機器は一度登録され、そこから検査に合格すると、まっすぐショーウィンドウへと進みます。品質管理は、誰も未検査の機器を買わないことを保証し、四つ目の原則は、安全に関わるステップでのうっかりミスから守ります。

## 勤怠記録、チーム、そして Hirn:社内業務

**何をするのか:** プラットフォームは日々の業務も担います。勤怠記録は、誰がいつ働いたかを、残高、報告、承認フローとともに記録します。チーム構造は、誰がどのグループで協力し、いま何に取り組んでいるかを映し出します。そして「Hirn」は、何もないところから当て推量するのではなく、Revamp-IT 自身の知識に基づいて答える RAG ベースの AI アシスタントです。

**どの問題を解くのか:** 非営利団体は、時間の乏しいボランティアと従業員によって成り立っています。そのうちの管理、探索、連携に消えていく分が少ないほど、本質のためにより多くが残ります — 機器を修理し、人々に届けることのためにです。アシスタントは、まず適切な人を見つけて尋ねる必要なく、社内の知識を呼び出せるものにします。

## 規模感

「沈黙する3つのシステム」から、相当な奥行きを持つひとつのプラットフォームが生まれました。その規模を示すいくつかの数値です:

| 指標 | 値 |
|---|---|
| API エンドポイント | 319 |
| ページ | 224 |
| 管理モジュール | 33 |
| データベーステーブル | ~130 |
| 言語 | 8 |
| コミット | 2'361 |

8つの言語は、私たちにとってボーナスではなく、使命の一部です:スイスは多言語であり、私たちのリーチは国際的で、手頃な技術は誰も言語の壁でつまずかせるべきではありません。

## その背後の技術

ボンネットの下には、現代的で意図をもって選ばれたスタックがあります。フロントエンドとプログラミングインターフェースは **Next.js 16** と **TypeScript** の上で動きます。データは **PostgreSQL** に置かれ、ORM の **Drizzle** を通してアクセスされるので、私たちの型はスキーマから直接導かれ、二重に保守する必要がありません。ログインは **NextAuth v5** が担います。マーケットプレイス全体の検索は **Meilisearch** が駆動します。決済は **Payrexx** が処理し、画像は **Cloudflare R2** に置かれ、**Sentry** はユーザーが報告しなければならなくなる前にエラーを私たちに知らせます。

どの個別の技術よりも重要なのは、その背後にある原則です:あらゆる情報についての単一の真実の源。ひとつの機器、ひとつの登録、販売まで一緒に進むひとつのレコード。まさに、3つの旧システムに欠けていたものです。

## 結論:3つの代わりに、ひとつの場所

かつて Joomla のウェブサイト、Shopware のショップ、Kivitendo の ERP が並んで動き、データが手作業で溝を越えて運ばれていたところに、いまはひとつのまとまったプラットフォームが立っています。それは、技術を手放す人や必要とする人と、修理する技術者を結びつけ — 同時に社内業務のすべてを担います。もはやサイロはなく、三重のデータ入力もなく、複数のウェブサイトを横断する探索もありません。

いちばん良いのは、これが単なる計画ではないということです。プラットフォームは稼働しており、**revampit.orangecat.ch** で検証できます。立ち寄って、マーケットプレイスを覗き、私たちの CO₂ 計算を確かめて — そして古い機器に新しい命を与えてください。
